HTML: Hyper Text Markup Language

  • Creates the structure of the webpage and is made up of elements that work together to provide content and basic framework.
  • Elements types include text, images, header, lists, page divisions, meta related, and semantic.
  • Attributes are inside open tags and also defined in beginning element.
  • The head element contains information about the webpage.
  • The head element is the opening element and contains metadata. Metadata is data about the webpage that is not seen by the user. It contains information about accessibility, search engines, and performance.
  • The head element contains title bar and language setting.
  • The body element represents the visible content shown to the user.
  • The body element has two different types of elements: Semantic and non-semantic.
  • Semantic elements are perferred and important because they make your HTML easier to read by other developers, browser, and accessibility to a wide variety of users.
  • Non-semantic elements are a simple solution to lay out problems. Example: div to create a division in page or span to create an inline division.
  • Headers elements are numbered 1-6, which refers to the level of the heading.
  • The paragraph element wraps text and the letter p in arrows is used.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch name: creates a new branch and switches to it
  • git add -A: add changed files into your working directory to the staging area of git
  • git commit -m "comment": commits changes to your repo. -m flag refers to a message attached.
  • git pull origin main: syncs local branch with the remote branch on Github
  • git push origin "branch": pushes changes to github
